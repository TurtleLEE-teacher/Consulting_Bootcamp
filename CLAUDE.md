# CLAUDE.md - 개발 가이드라인

이 문서는 슬라이드 제작 시 일관성을 유지하기 위한 가이드라인입니다.

---

## 1. 톤앤매너

### 1.1 전체 톤
| 항목 | 가이드 |
|------|--------|
| **어조** | 전문적이면서 친근함, 강의체 |
| **존칭** | 해요체 사용 ("~합니다" ❌ "~해요" ✅) |
| **난이도** | 컨설팅 초심자도 이해 가능한 수준 |
| **예시** | 실제 프로젝트 상황을 연상할 수 있는 구체적 예시 |

### 1.2 슬라이드별 톤
| 슬라이드 유형 | 톤 |
|--------------|-----|
| 개념 설명 | 명확하고 간결하게 |
| 비교/대조 | 차이점을 선명하게 부각 |
| 예시/사례 | 스토리텔링, 몰입감 |
| 요약 | 핵심만 짧게 |

### 1.3 피해야 할 표현
- 지나친 전문용어 남발 (약어는 처음 등장 시 풀어서 설명)
- "~입니다", "~습니다" 딱딱한 문어체
- 불필요한 영어 표현 (한글로 충분한 경우)
- 모호한 표현 ("대략", "약간", "어느 정도")

## 2. 디자인 시스템

### 2.1 색상 팔레트

```css
/* Primary - 메인 강조색 */
--primary: #2563EB;        /* Blue 600 */
--primary-light: #3B82F6;  /* Blue 500 */
--primary-dark: #1D4ED8;   /* Blue 700 */

/* Secondary - 보조색 */
--secondary: #10B981;      /* Emerald 500 */
--secondary-light: #34D399;

/* Accent - 포인트 */
--accent: #F59E0B;         /* Amber 500 */
--accent-alt: #EF4444;     /* Red 500 - 주의/중요 */

/* Neutral - 텍스트, 배경 */
--text-dark: #1F2937;      /* Gray 800 */
--text-light: #6B7280;     /* Gray 500 */
--bg-light: #F9FAFB;       /* Gray 50 */
--bg-white: #FFFFFF;

/* A-E-D-B-D 단계별 색상 */
--assess: #8B5CF6;         /* Violet 500 */
--envision: #06B6D4;       /* Cyan 500 */
--design: #10B981;         /* Emerald 500 */
--build: #F59E0B;          /* Amber 500 */
--deploy: #EF4444;         /* Red 500 */
```

### 2.2 타이포그래피

```css
/* 폰트 패밀리 */
--font-main: 'Pretendard', -apple-system, sans-serif;
--font-code: 'JetBrains Mono', monospace;

/* 폰트 사이즈 */
--title-size: 48px;        /* 슬라이드 제목 */
--subtitle-size: 32px;     /* 섹션 제목 */
--body-size: 24px;         /* 본문 */
--caption-size: 18px;      /* 캡션, 주석 */

/* 폰트 웨이트 */
--weight-bold: 700;
--weight-semibold: 600;
--weight-regular: 400;
```

### 2.3 레이아웃 규칙

| 항목 | 값 |
|------|-----|
| 슬라이드 비율 | 16:9 |
| 상하 여백 | 60px |
| 좌우 여백 | 80px |
| 요소 간 간격 | 24px |
| 최대 줄 수 | 6줄 |
| 글머리 최대 | 5개 |

### 2.4 비주얼 요소

#### 아이콘 스타일
- **스타일**: Line icons (Outline)
- **크기**: 48px (대), 32px (중), 24px (소)
- **색상**: Primary 또는 단계별 색상
- **소스**: Heroicons, Lucide Icons

#### 다이어그램
- **도구**: Mermaid.js (플로우차트, 시퀀스)
- **스타일**: 둥근 모서리, 그림자 없음
- **색상**: 단계별 색상 팔레트 준수

## 3. 슬라이드 작성 규칙

### 3.1 제목 슬라이드
```html
<section class="title-slide">
  <h1>섹션 제목</h1>
  <p class="subtitle">부제목 또는 설명</p>
</section>
```

### 3.2 내용 슬라이드
```html
<section>
  <h2>슬라이드 제목</h2>
  <ul>
    <li>핵심 포인트 1</li>
    <li>핵심 포인트 2</li>
    <li>핵심 포인트 3</li>
  </ul>
</section>
```

### 3.3 2분할 레이아웃
```html
<section class="two-column">
  <div class="left">
    <h3>왼쪽 제목</h3>
    <p>내용</p>
  </div>
  <div class="right">
    <h3>오른쪽 제목</h3>
    <p>내용</p>
  </div>
</section>
```

### 3.4 다이어그램 슬라이드
```html
<section>
  <h2>A-E-D-B-D 프로세스</h2>
  <div class="mermaid">
    flowchart LR
      A[ASSESS] --> E[ENVISION]
      E --> D[DESIGN]
      D --> B[BUILD]
      B --> DP[DEPLOY]
  </div>
</section>
```

## 4. 콘텐츠 가이드

### 4.1 슬라이드당 콘텐츠 양
| 유형 | 권장량 |
|------|--------|
| 텍스트 슬라이드 | 최대 50단어 |
| 리스트 슬라이드 | 최대 5개 항목 |
| 비교 슬라이드 | 2~3개 비교 대상 |
| 다이어그램 | 1개 메인 + 설명 텍스트 |

### 4.2 핵심 메시지 원칙
- **1 슬라이드 = 1 메시지**: 하나의 슬라이드에서 전달할 핵심은 1개
- **So What?**: 매 슬라이드마다 "그래서 뭐?"에 대한 답이 있어야 함
- **시각화 우선**: 텍스트보다 다이어그램/차트로 표현 가능하면 시각화

### 4.3 전환 문구
슬라이드 간 자연스러운 흐름을 위해 전환 문구 사용:

| 상황 | 예시 |
|------|------|
| 다음 주제로 | "이제 ~ 살펴볼게요" |
| 심화 | "좀 더 자세히 보면" |
| 예시 제시 | "예를 들어" |
| 요약 | "정리하면" |
| Part 전환 | "지금까지 ~를 봤고, 이제부터 ~" |

## 5. PDF 변환 체크리스트

### 5.1 변환 전 확인
- [ ] 모든 슬라이드가 16:9 비율에 맞는지
- [ ] 텍스트가 여백을 침범하지 않는지
- [ ] 다이어그램이 제대로 렌더링되는지
- [ ] 폰트가 모두 로드되었는지

### 5.2 변환 설정
```
URL: http://localhost:3000?print-pdf
브라우저: Chrome 권장
설정:
  - 용지: A4 또는 Letter
  - 여백: 없음
  - 배경 그래픽: 체크
  - 가로 방향: 체크
```

### 5.3 변환 후 확인
- [ ] 모든 페이지가 잘림 없이 출력되었는지
- [ ] 페이지 번호가 올바른지
- [ ] 색상이 제대로 나왔는지
- [ ] 파일 크기가 적정한지 (권장: 10MB 이하)
