## 사례 1: 제조사 재고 최적화

**배경 (Situation):**
- **산업**: 자동차 부품 제조
- **규모**: 연매출 5,000억원, 임직원 1,200명
- **문제**: 과도한 재고로 인한 운전자본 압박

**과제 (Complication):**
- **현재 상태**: 재고 회전일 80일 (업계 평균 45일)
- **목표 상태**: 재고 회전일 30일 이하
- **제약조건**: 생산 중단 없이 개선, 6개월 내 완료
- **경영 임팩트**: 묵혀있는 현금 600억원

**접근법 (Answer):**

1. **ASSESS (3주)**: 현황 분석
   - 품목별 재고 현황 데이터 수집 (8,500개 SKU)
   - ABC 분석으로 핵심 품목 120개 선정
   - 재고 발생 원인 5가지 도출
     * 과다 안전재고 (40%)
     * 부정확한 수요예측 (25%)
     * 긴 리드타임 (20%)
     * 대량 발주 (10%)
     * 불량재고 (5%)

2. **ENVISION (2주)**: 개선 방향 수립
   - Quick Win: 불량재고 즉시 처리 → 50억 회수
   - Major Project: SCM 프로세스 재설계
   - 목표: 80일 → 30일 (62% 감축)

3. **DESIGN (4주)**: 상세 설계
   - 안전재고 공식 재정의 (업계 벤치마크)
   - 수요예측 모델 구축 (12개월 이력 데이터 활용)
   - 협력사 리드타임 단축 협상
   - VMI(Vendor Managed Inventory) 도입 설계

4. **BUILD (10주)**: 시스템 구축
   - ERP 재고관리 모듈 고도화
   - 수요예측 자동화 (Python 스크립트)
   - 발주 자동화 시스템
   - 협력사 EDI 연동

5. **DEPLOY (3주)**: 파일럿 & 확산
   - 핵심 120개 품목 파일럿 (2주)
   - 전체 8,500개 품목 확산 (1주)

**결과 (Result):**
- **Before**: 재고 회전일 80일, 재고자산 800억원
- **After**: 재고 회전일 30일, 재고자산 300억원
- **임팩트**:
  * 운전자본 500억원 절감 (-62%)
  * 창고 임대료 연 15억원 절감
  * 재고 관리 인력 8명 → 3명
- **소요기간**: 5.5개월
- **프로젝트 비용**: 3억원
- **ROI**: 16,600% (500억 / 3억)

**핵심 인사이트:**
> "재고는 숨어있는 현금이다. 데이터 기반 의사결정으로 6개월 만에 500억을 찾아냈다."

**슬라이드 적용:**
- P1-004 (컨설팅의 가치): ROI 16,600% 수치 강조
- P2-A-008 (데이터 수집): ABC 분석 사례로 활용
- P2-E-004 (Quick Win): 불량재고 즉시 처리 50억 사례

## 사례 2: 이슈 도출 프로세스

**배경 (Situation):**
- **산업**: 유통/리테일
- **규모**: 전국 250개 매장, 연매출 1.2조원
- **문제**: "모든 게 문제다" - 산만한 문제 정의

**과제 (Complication):**
- **현재 상태**: 경영진이 나열한 문제 100개+
  * 매출 감소
  * 재고 과다
  * 인건비 상승
  * 온라인 경쟁 심화
  * 고객 이탈
  * ... (95개 더)
- **목표 상태**: 집중할 핵심 이슈 3-5개 선정
- **제약조건**: 3주 내 우선순위화 완료

**접근법 (Answer):**

**Step 1: 프로세스 매핑 (1주)**
- End-to-End 가치사슬 가시화
  ```
  상품기획 → 소싱 → 입고 → 진열 → 판매 → 재고관리 → 반품
  ```
- 각 단계별 문제 분류
- 결과: 100개 → 7개 프로세스별 그룹화

**Step 2: Issue Tree 구조화 (1주)**
```
매출 감소 (-15% YoY)
├─ 객단가 하락 (-8%)
│  ├─ 프로모션 남발 → 정상가 구매 감소
│  └─ 저가 상품 비중 증가 → 믹스 악화
├─ 방문객 수 감소 (-5%)
│  ├─ 온라인 전환 → 오프라인 매장 방문 감소
│  └─ 입지 경쟁력 약화 → 신규 몰 오픈
└─ 구매전환율 하락 (-3%)
   ├─ 재고 부족 → 품절로 구매 포기
   └─ 서비스 품질 → 직원 숙련도 부족
```

**Step 3: 2x2 매트릭스 우선순위화 (1주)**

| | 실행 쉬움 | 실행 어려움 |
|---|---|---|
| **임팩트 큼** | **Q1: Quick Win**<br>✅ 프로모션 정책 개선<br>✅ 재고 보충 체계 개선 | **Q2: Major Project**<br>🔲 온라인 통합 전략<br>🔲 상품믹스 재정립 |
| **임팩트 작음** | **Q3: Fill-in**<br>○ 서비스 교육<br>○ 진열 개선 | **Q4: 제외**<br>✗ 입지 변경<br>✗ 브랜드 리뉴얼 |

**핵심 이슈 3개 선정:**
1. 프로모션 남발로 인한 객단가 하락 (-8%)
2. 재고 부족으로 인한 구매전환율 하락 (-3%)
3. 온라인 전환 대응 미흡 (-5% 방문객 감소)

**결과 (Result):**
- **Before**: 산만한 100개 문제, 우선순위 불명
- **After**: 데이터 기반 핵심 이슈 3개
- **임팩트**:
  * 프로젝트 범위 명확화 → 3개월 일정 단축
  * 실행 집중도 향상 → Quick Win부터 즉시 착수
  * 경영진 의사결정 속도 3배 향상
- **소요기간**: 3주 (ASSESS 단계)

**핵심 인사이트:**
> "100개 문제를 동시에 풀 순 없다. 데이터로 3개를 찾아내고, Quick Win부터 시작하라."

**슬라이드 적용:**
- P2-A-001 (ASSESS 타이틀): "100개 문제 중 3개를 찾아내는 법"
- P2-A-006 (Issue Tree): 실제 매출 감소 Issue Tree 다이어그램
- P2-A-009 (우선순위화): 2x2 매트릭스 실제 사례

## 사례 3: 개선 우선순위화 (Quick Win 전략)

**배경 (Situation):**
- **산업**: 제약/바이오
- **규모**: 연매출 3,000억원, R&D 인력 400명
- **문제**: 신약 개발 프로세스 비효율

**과제 (Complication):**
- **현재 상태**: 임상시험 승인 기간 평균 180일 (경쟁사 120일)
- **목표 상태**: 승인 기간 120일 이하로 단축
- **제약조건**:
  * 규제 준수 필수 (FDA, MFDS)
  * 대규모 투자 불가 (예산 10억 이하)
  * 6개월 내 개선 효과 가시화

**접근법 (Answer):**

**ENVISION: 개선 아이디어 30개 도출**

브레인스토밍으로 30개 개선안 수집:
- 전자문서 시스템 도입
- AI 기반 문서 자동 검토
- 승인 단계 간소화
- 협력사 선정 프로세스 개선
- 템플릿 표준화
- ... (25개 더)

**Quick Win 매트릭스로 우선순위화**

```
             영향도 높음
                 ↑
      Q2: Major Project  │  Q1: Quick Win
      - 전자문서 시스템   │  ✅ 템플릿 표준화 (즉시 시작)
      - AI 자동검토      │  ✅ 체크리스트 개선
─────────────────────────┼─────────────────────────→ 실행 쉬움
      Q4: 제외           │  Q3: Fill-in
      - 조직개편         │  - 교육 프로그램
      - 외주 전환        │  - 매뉴얼 정비
                 ↓
             영향도 낮음
```

**Quick Win 3가지 (투자 0원, 2주 내 실행)**

1. **문서 템플릿 표준화**
   - 문제: 매번 새로 작성 → 검토 시간 과다
   - 해결: 승인 받은 베스트 케이스 10개를 템플릿화
   - 효과: 작성 시간 40시간 → 8시간 (80% 단축)

2. **체크리스트 정교화**
   - 문제: 누락 항목으로 반려 → 재작업
   - 해결: 반려 사유 TOP 10을 체크리스트에 포함
   - 효과: 1차 반려율 40% → 15% (62% 감소)

3. **병렬 검토 프로세스**
   - 문제: 순차 검토 (법무 → 의학 → 규제)
   - 해결: 독립적 항목은 병렬 검토
   - 효과: 검토 기간 45일 → 30일 (33% 단축)

**Major Project (6개월, 예산 8억)**

4. **전자문서 시스템 (ERP 연동)**
   - 수기 입력 제거
   - 자동 알림/추적
   - 예상 효과: 추가 30일 단축

**결과 (Result):**

| 구분 | Before | Quick Win 후 | Major Project 후 |
|------|--------|--------------|------------------|
| 승인 기간 | 180일 | 135일 (-25%) | 105일 (-42%) |
| 투자 비용 | - | 0원 | 8억원 |
| 소요 시간 | - | 2주 | 6개월 |

**임팩트:**
- Quick Win으로 즉시 45일 단축 → 경영진 신뢰 확보
- 신뢰 기반으로 8억 예산 승인 → Major Project 추진
- 최종적으로 경쟁사 대비 15일 빠른 시장 진입

**핵심 인사이트:**
> "큰 프로젝트는 신뢰가 필요하다. Quick Win으로 먼저 성과를 보여주고, 그 다음 Major Project를 요구하라."

**슬라이드 적용:**
- P2-E-001 (ENVISION 타이틀): Quick Win의 힘
- P2-E-005 (우선순위 매트릭스): 실제 30개 아이디어 분류
- P2-E-007 (Quick Win 전략): 0원 투자, 2주 실행, 25% 개선

## 사례 4: BPR 적용 사례 (프로세스 재설계)

**배경 (Situation):**
- **산업**: 금융/보험
- **규모**: 손해보험사, 연 보험료 수입 8조원
- **문제**: 보험금 지급 프로세스 지연 → 고객 불만 급증

**과제 (Complication):**
- **현재 상태**:
  * 평균 지급 소요일: 14일 (업계 평균 7일)
  * 고객 불만율: 28% (업계 평균 12%)
  * NPS(순추천지수): -15 (경쟁사 +20)
- **목표 상태**:
  * 평균 지급 소요일: 5일 이하
  * 고객 불만율: 10% 이하
- **제약조건**:
  * 보험사기 검증 프로세스 필수 유지
  * 기존 인력 구조 유지 (감원 없이)

**접근법 (Answer):**

**DESIGN: BPR (Business Process Reengineering)**

**1. As-Is 프로세스 분석 (현재 7단계, 14일)**

```
[접수] → [1차 검토] → [서류요청] → [대기] → [2차 검토] → [승인] → [지급]
  1일      2일         1일       5일      3일       1일     1일

  ↓ 문제점 분석

- 서류 요청 후 고객 대기: 5일 (전체의 36%)
- 중복 검토: 1차 검토와 2차 검토에서 동일 항목 재확인
- 수작업 입력: 서류 스캔 후 수기 입력 (오류율 8%)
- 승인 대기: 담당자별 결재 권한 불명확
```

**2. To-Be 프로세스 설계 (신규 3단계, 5일)**

```
[원클릭 접수] → [AI 자동심사] → [승인/지급]
      1일           3일            1일

핵심 변화:
✅ 서류 요청 단계 제거 → OCR로 자동 추출
✅ 1차/2차 검토 통합 → AI 리스크 스코어링
✅ 승인 자동화 → 5천만원 이하 자동 승인
```

**3. 변화 설계 상세**

| 기존 (As-Is) | 문제점 | 개선 (To-Be) | 효과 |
|-------------|--------|-------------|------|
| 고객이 서류 우편 발송 | 5일 소요 | 모바일 사진 업로드 | 즉시 |
| 직원이 서류 수기 입력 | 2일, 오류 8% | OCR 자동 인식 | 10분, 오류 0.5% |
| 1차 검토 (주니어) | 2일 | AI 리스크 분석 | 3시간 |
| 2차 검토 (시니어) | 3일 | 고위험만 사람 검토 | 15% 케이스만 |
| 결재 승인 대기 | 1일 | 자동 승인 (85%) | 즉시 |

**4. 기술 스택**

- **OCR**: Google Cloud Vision API
- **AI 심사**:
  * 과거 3년 100만건 데이터 학습
  * 사기 확률 0-100점 스코어링
  * 80점 이상 → 자동 승인
  * 20점 이하 → 자동 거절
  * 21-79점 → 사람 검토
- **자동화**: RPA (UiPath)

**결과 (Result):**

| 지표 | Before | After | 개선율 |
|------|--------|-------|--------|
| 평균 지급일 | 14일 | 5일 | -64% |
| 고객 불만율 | 28% | 9% | -68% |
| NPS | -15 | +25 | +40p |
| 처리 인력 | 120명 | 120명 | 0% (재배치) |
| 시간당 처리건수 | 2건 | 8건 | +300% |

**부가 효과:**
- 인력 재배치: 단순 입력 120명 → 고객상담 80명 + 사기조사 40명
- 보험사기 적발률: 2% → 5% (AI 정확도 향상)
- 고객 만족도 1위 달성 (업계 내)

**소요기간 & 비용:**
- DESIGN: 6주 (프로세스 재설계 + 시스템 설계)
- BUILD: 16주 (AI 모델 개발 + RPA 구축)
- 총 비용: 18억원
- ROI: 인건비 절감 불가 → NPS 상승으로 신규 고객 유입 (연 2,000억 보험료 증가)

**핵심 인사이트:**
> "BPR은 단순 자동화가 아니다. 프로세스를 '재설계'하는 것이다. 7단계를 3단계로 줄이니, 14일이 5일이 되었다."

**슬라이드 적용:**
- P1-013 (BPR 사례): As-Is 7단계 → To-Be 3단계 다이어그램
- P2-D-001 (DESIGN 타이틀): "프로세스를 버려라" 메시지
- P2-D-008 (BPR 방법론): 실제 적용 사례로 설명

## 사례 5: ERP 구축 (디지털 전환)

**배경 (Situation):**
- **산업**: 화학/소재
- **규모**: 연매출 2조원, 전국 5개 공장
- **문제**: 레거시 시스템 노후화 → 실시간 의사결정 불가

**과제 (Complication):**
- **현재 상태**:
  * 공장별 독립 시스템 (5개 서로 다른 시스템)
  * 수작업 데이터 통합: 매월 말 5일 소요
  * 재무제표 마감: 15일 소요 (상장사 평균 7일)
  * Excel 기반 의사결정: 데이터 신뢰도 낮음
- **목표 상태**:
  * 전사 통합 ERP 구축
  * 실시간 재무/생산 현황 조회
  * 재무제표 마감 3일 이내
- **제약조건**:
  * 생산 중단 불가 (24시간 가동)
