## 📋 Phase 2: 스토리라인 재구성

### 목표

SCQA (Situation-Complication-Question-Answer) 프레임워크 적용

### Part 1 재구성안

#### 현재 구조 (문제점)
```
1. 오프닝
2. 컨설팅이란?
3. 산업 구조
4. PI vs BPR
5. A-E-D-B-D 소개
6. 5C 역량
7. 커리어
```
→ **순차적 정보 나열, 긴장감 없음**

#### 개선 구조 (SCQA 적용)

**Act 1: 상황 + 문제 제기 (10분, 5장)**
```
S: 타이틀 → "왜 컨설턴트가 되려고 하는가?"
C: 데이터로 보는 컨설팅 시장
   - 국내 시장 CAGR 15% (2020-2025)
   - MBB 신입 연봉 $120K+
   - 但, 합격률 2%, 퇴사율 30%
Q: "성공하려면 무엇이 필요한가?"
```

**Act 2: 컨설팅 본질 이해 (15분, 8장)**
```
A1: 컨설팅이란 무엇인가?
   - 정의: 6개월에 3년치 성과
   - 실제 사례: 재고 80일→30일 (제조사 A)
   - 가치: ROI 300%

A2: 컨설팅 시장 구조
   - MBB vs Big4 vs Boutique
   - 전략 vs 운영 vs IT
   - 프로젝트 유형별 특징
```

**Act 3: 방법론과 역량 (20분, 10장)**
```
A3: A-E-D-B-D 방법론
   - 왜 이 순서인가?
   - 실제 프로젝트 타임라인
   - 각 단계 핵심 산출물

A4: 5C 역량 모델
   - McKinsey가 뽑는 사람
   - 합격자 vs 불합격자 차이점
   - 역량 개발 로드맵
```

**Act 4: 커리어와 미래 (15분, 8장)**
```
A5: 컨설턴트의 삶
   - 하루 일과 (데이터 기반)
   - 커리어 트랙 (Associate → Partner)
   - Up or Out 문화의 양면

A6: Exit 옵션
   - 기업 전략실 (연봉, 워라밸)
   - 스타트업 (성장, 스톡옵션)
   - 창업 (자율, 리스크)
```

### Part 2 (각 단계) 재구성안

**모든 단계에 SCQA 적용**

예: ASSESS 단계
```
S: "프로젝트 Day 1, 클라이언트는 혼란스럽다"
   → 100개의 문제, 무엇부터?

C: 잘못된 접근법
   → 가장 큰 문제부터? ✗
   → 가장 쉬운 것부터? ✗
   → 경영진 말대로? ✗

Q: "어떻게 진짜 문제를 찾는가?"

A: ASSESS 3단계
   1. 프로세스 매핑 → 가시화
   2. Issue Tree → 구조화
   3. 2x2 Matrix → 우선순위화

   실제 사례: 제조사 재고 문제 해결 프로세스
```
